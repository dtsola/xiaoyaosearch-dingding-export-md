# 精简MRD：小遥搜索钉钉导出工具

## 1. 核心问题（一句话）
**解决什么问题？**
- 为使用钉钉进行知识管理的用户提供一个跨平台命令行工具，解决钉钉官方不支持批量导出、知识库归档困难、数据迁移成本高等问题。
- **核心目的：** 将钉钉文档导出为 Markdown 格式，便于对接 [小遥搜索](https://github.com/dtsola/xiaoyaosearch) 实现本地 AI 智能搜索。

## 2. 目标用户（具体画像）

### 人群特征
- **小遥搜索用户** - 已在使用小遥搜索，希望将钉钉文档导出后接入本地 AI 搜索，实现「听懂你的话、看懂你的图，用 AI 找到本地任何文件」
- **知识管理爱好者** - 有大量个人文档存储在钉钉，希望定期备份到本地
- **自由职业者/Indie Hacker** - 使用钉钉协作，需要将内容迁移到 Obsidian、Logseq 等本地笔记工具
- **创业团队 IT 运维** - 负责企业钉钉知识库的定期备份和合规归档
- **技术文档维护者** - 需要将 API 文档、技术规范导出到 Git 仓库进行版本控制
- **DevOps 工程师** - 希望将导出工具集成到 crontab 或 CI/CD 流程实现自动化备份

### 痛点场景
1. **无法批量导出** - 钉钉官方只能单篇手动导出，大量文档导出耗时极长
2. **知识库归档困难** - 企业知识库需要定期备份以满足合规要求，但缺乏自动化方案
3. **数据迁移成本高** - 切换知识管理工具时，无法将历史文档完整迁移
4. **担心数据丢失** - 云端文档存在平台关停、账号封禁等风险，需要本地备份
5. **跨平台需求** - Windows/macOS/Linux 用户都需要统一的导出方案

### 当前替代方案
- **手动导出** - 逐篇打开文档复制内容，效率极低
- **钉钉官方 MCP 工具** - 仅服务于钉钉 AI 生态，无法将数据导出到本地对接第三方工具（如小遥搜索）
- **第三方付费服务** - 价格昂贵，且存在数据安全风险
- **放弃迁移** - 继续留在钉钉，承担数据锁定风险

## 3. 解决方案（核心功能）

**MVP三个核心功能：**

1. **单个/批量文档导出**
   - 支持通过文档 ID 或 URL 导出单篇钉钉文档
   - 支持从文件读取文档 ID 列表，批量导出多个文档
   - 自动下载文档中的图片和附件
   - 导出为标准 Markdown 格式，便于迁移到其他工具，**更可直接对接小遥搜索实现本地 AI 智能检索**

2. **知识库导出**
   - 支持导出整个钉钉知识库
   - 保持原有目录结构和层级关系
   - 递归导出所有子文档
   - 支持增量导出（仅导出有更新的文档）

3. **命令行配置管理**
   - 交互式初始化配置（钉钉应用 AppKey、AppSecret）
   - 支持多配置文件（不同账号/企业）
   - 环境变量支持
   - 并发控制（避免触发 API 限流）

## 4. 竞品情况

| 竞品 | 优势 | 劣势 | 差异化机会 |
|------|------|------|-----------|
| 钉钉官方导出 | 官方支持、格式兼容 | 只能单篇手动导出、无法批量、无法导出知识库 | 批量导出、自动化、知识库完整导出 |
| 钉钉官方 MCP 工具 | 官方出品、集成 AI 能力 | 主要服务于钉钉 AI 生态，无法导出到本地对接小遥搜索 | 专注本地化导出，对接小遥搜索实现私有 AI 搜索 |
| 飞书导出工具（自家） | 功能完善、用户认可 | 仅支持飞书平台 | 钉钉用户是一个独立的大市场 |
| 第三方爬虫工具 | 可能支持多平台 | 不稳定、易失效、安全风险、收费 | 使用官方 API、稳定可靠、完全免费 |
| 浏览器插件 | 安装简单 | 无法批量、无法自动化、不跨平台 | 命令行工具可集成到自动化流程 |

## 5. 商业模式

- **变现方式：** 完全免费（引流工具）
  - **核心价值：** 将钉钉文档导出后可无缝对接小遥搜索，实现私有知识库的本地 AI 智能搜索
  - 作为 [小遥搜索](https://github.com/dtsola/xiaoyaosearch) 生态的扩展工具，完善「数据导入」能力
  - 吸引钉钉用户了解和使用小遥搜索主产品
  - 建立品牌认知和用户信任

- **预估成本：**
  - 开发成本：1-2 周开发时间（可复用飞书导出工具架构）
  - 维护成本：钉钉 API 更新时需要适配（预计低频）
  - 运营成本：GitHub Pages 托管文档、开源社区维护

- **预估收入：** 间接收益
  - 引流到小遥搜索的潜在付费用户
  - 建立技术品牌，吸引合作机会

## 6. 风险评估

- **技术风险：**
  - 钉钉 API 权限申请可能比飞书更严格
  - 钉钉文档格式转换可能存在兼容性问题
  - **缓解措施：** 参考飞书工具已验证的技术方案，逐步测试

- **市场风险：**
  - 钉钉用户对命令行工具的接受度可能较低
  - 钉钉官方可能推出批量导出功能
  - **缓解措施：** 提供详细的使用文档和截图，快速迭代

- **时间风险：**
  - 钉钉 API 学习曲线
  - **缓解措施：** 复用飞书工具的代码架构和设计模式
