# 精简MRD：小遥搜索钉钉导出工具

## 1. 核心问题（一句话）

**钉钉用户缺乏一款设计精美、体验流畅的文档导出工具，无法高效地将钉钉文档批量导出为本地可用的格式。**

---

## 2. 目标用户（具体画像）

### 2.1 用户群体

| 用户类型 | 人群特征 | 痛点场景 | 当前替代方案 |
|---------|---------|---------|-------------|
| **知识管理爱好者** | 重视数据主权，习惯本地备份 | 云端文档可能丢失，担心平台关停 | 手动逐个下载，效率极低 |
| **自由职业者** | 需要在多个工具间协作迁移 | 需要迁移到 Obsidian/Logseq 等本地笔记 | 复制粘贴导致格式丢失 |
| **创业团队 IT 运维** | 负责企业数字资产的管理 | 企业知识库需要定期备份归档 | 缺乏自动化备份工具 |
| **技术文档维护者** | 维护API文档、技术规范等 | 需要导出到Git进行版本控制 | 无批量导出能力 |
| **DevOps 工程师** | 追求自动化和效率提升 | 希望集成到定时任务/CI流程 | 现有工具不支持集成 |
| **学生用户** | 使用多种设备和操作系统 | 需要在 Windows/macOS 间切换 | 工具兼容性差 |

### 2.2 用户痛点总结

1. **效率问题** - 手动下载大量文档耗时耗力
2. **格式问题** - 复制粘贴导致格式丢失、图片缺失
3. **结构问题** - 批量导出后无法保持原有目录结构
4. **体验问题** - 现有工具界面简陋、操作复杂、稳定性差
5. **安全焦虑** - 担心云端数据丢失或平台政策变化

---

## 3. 解决方案（核心功能）

### MVP三个核心功能：

#### 3.1 单个文档导出
- 支持钉钉文档（.adoc）导出为 **.md / .docx / .pdf**
- 支持钉钉表格（.axls）导出为 **.xlsx**
- 支持钉钉白板（.adraw）导出为 **.jpg**
- 支持钉钉脑图（.amind）导出为 **.jpg**
- 自动下载文档中的图片和附件

#### 3.2 批量文档导出
- 支持在钉钉文档页面一键打开导出面板
- 可视化文档树，支持勾选多个文档
- 批量导出时保持原有目录结构
- 实时显示导出进度和状态

#### 3.3 知识库导出
- 支持完整知识库一键导出
- 递归导出所有子文档和文件夹
- 保持知识库的层级结构
- 支持选择性导出（排除不需要的内容）

## 5. 商业模式

### 5.1 变现方式

**免费开源，作为小遥搜索生态的引流工具**

- 本工具不直接收费，完全免费使用
- 通过提供优质工具吸引用户了解小遥搜索主产品
- 在工具页面适当展示小遥搜索的介绍和链接
- 用户使用导出的 Markdown 文件后，可能需要本地搜索工具 → 引流至小遥搜索

### 5.2 预估成本

| 成本类型 | 说明 |

| **开发成本** | 前端开发时间（约 2-3 周） |

| **运营成本** | Chrome扩展商店费用（一次性 $5） |
| **服务器成本** | 无（纯前端方案，无需服务器） |

### 5.3 预估收入

**间接收入**：通过引流至小遥搜索主产品，实现商业化转化
