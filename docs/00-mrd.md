# 小遥搜索钉钉导出工具 - MRD

## 1. 核心问题（一句话）
**解决什么问题？**
钉钉文档缺乏批量导出功能，用户无法高效地备份和迁移自己的知识库文档。

## 2. 目标用户（具体画像）

### 人群特征

- 👨‍💻 **知识管理爱好者** - 希望将云端文档备份到本地，防止数据丢失，离线也能随时查看
- 💼 **自由职业者/独立开发者** - 需要将钉钉内容迁移到 Obsidian、Logseq、Notion 等本地笔记工具
- 🏢 **创业团队 IT 运维** - 负责企业知识库的定期备份和合规归档，满足企业审计要求
- 📝 **技术文档维护者** - 需要将 API 文档、技术规范导出到本地进行版本控制
- 🎓 **学生用户** - 在 Windows/macOS 等不同平台上都需要使用，跨平台兼容性需求高

### 痛点场景

1. **📦 个人知识备份**
   - 珍贵的钉钉文档需要备份到本地，即使没有网络也能随时查看
   - 担心云端数据丢失风险，需要定期备份保障数据安全

2. **🔄 笔记工具迁移**
   - 导出为标准 Markdown/.docx/.pdf 等格式，轻松迁移到 Obsidian、Logseq、Notion 等其他知识管理工具
   - 需要保持原有目录结构，方便迁移后继续使用

3. **🏛️ 企业知识归档**
   - 批量导出整个钉钉知识库，保持原有目录结构
   - 满足企业合规和审计要求，定期归档备份

4. **📊 技术文档本地化**
   - 将 API 文档、技术规范、设计文档导出到本地
   - 实现离线查阅和版本管理，支持 Git 等版本控制工具

5. **🚀 离职文档交接**
   - 一键导出所有参与过的项目文档，快速完成工作资料交接
   - 避免数据丢失，确保工作成果完整保留

### 当前替代方案
| 方案 | 缺点 |
|------|------|
| 手动逐个下载 | 耗时长，目录结构丢失 |
| 钉钉官方导出 | 功能受限，不支持批量操作 |
| 第三方工具 | ding-doc-downloader 等项目缺乏维护，UI 体验差 |

## 3. 解决方案（核心功能）

**MVP三个核心功能：**
1. **一键批量导出**：支持递归选择文件夹，批量导出整个知识库
2. **多格式转换**：支持导出为 .docx/.md/.pdf/.xlsx/.jpg 等常用格式
3. **目录结构保持**：完整保留钉钉文档的文件夹层级结构

### 已实现功能详情

#### 支持的文档类型及导出格式
| 钉文档类型 | 可导出格式 |
|-----------|-----------|
| 文档 (.adoc) | .md, .docx, .pdf |
| 表格 (.axls) | .xlsx |
| 白板 (.adraw) | .jpg |
| 脑图 (.amind) | .jpg |
| 其他文件 | 原格式 |

#### 产品形态
- **浏览器插件**：Chrome/Edge 扩展
- **激活方式**：访问钉钉文档页面后点击插件图标
- **交互方式**：右下角悬浮卡片，实时显示下载进度

## 4. 竞品情况

| 竞品 | 优势 | 劣势 | 差异化机会 |
|------|------|------|-----------|
| ding-doc-downloader | 开源免费，功能完整 | 自研框架维护困难，UI 过时 | 现代化 Apple 风格 UI，持续维护 |
| 钉钉官方导出 | 官方支持，稳定可靠 | 不支持批量操作，格式受限 | 批量处理，格式丰富 |
| 网页抓取工具 | 通用性强 | 配置复杂，容易失效 | 专门优化钉钉场景，开箱即用 |
