# 精简MRD：小遥搜索钉钉导出工具

## 1. 核心问题（一句话）

**解决什么问题？**

钉钉用户无法方便地批量导出文档和知识库，缺乏跨平台的本地化导出工具，导致知识资产被锁定在钉钉平台，用户无法自由管理和备份自己的文档。

---

## 2. 目标用户（具体画像）

### 2.1 人群特征

| 用户类型 | 特征描述 | 比例 |
|---------|---------|------|
| **知识工作者** | 在钉钉上积累大量文档，需要定期备份和知识管理 | 50% |
| **独立开发者/自由职业者** | 多平台协作，需要将钉钉文档迁移到其他工具（Obsidian、Notion等） | 30% |
| **团队管理员** | 需要导出团队知识库进行归档或迁移 | 15% |
| **个人用户** | 离职/转岗前需要导出个人文档 | 5% |

### 2.2 痛点场景

1. **离职场景**：员工离职时，需要快速导出所有工作文档，但钉钉不支持批量导出
2. **知识迁移**：团队从钉钉迁移到其他平台，需要批量导出知识库
3. **本地备份**：担心钉钉账号异常或数据丢失，需要定期备份重要文档
4. **离线使用**：需要在无网络环境下查看和编辑文档
5. **格式转换**：需要将钉钉专有格式（.adoc）转换为通用格式（Markdown、PDF等）

### 2.3 当前替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **手动逐个导出** | 官方支持，稳定可靠 | 效率极低，无法批量操作 |
| **ding-doc-downloader** | 支持批量导出，功能完善 | 只能在浏览器中运行，无法自动化；缺乏命令行支持 |
| **第三方SaaS服务** | 无需安装 | 隐私风险，需要付费，依赖在线服务 |
| **放弃导出** | 无成本 | 知识资产永久丢失 |

---

## 3. 解决方案（核心功能）

**产品形态**：浏览器插件（Chrome/Edge扩展，跨平台支持）

**技术栈**：基于Node.js生态，使用现代前端框架开发

### MVP三个核心功能：

#### 1. 插件配置面板

用户通过浏览器插件的配置面板进行设置：
- 导出格式选择（Markdown、DOCX、PDF、XLSX、JPG等）
- 导出路径设置
- 文件命名规则配置
- 批量导出参数配置

**价值**：直观的可视化配置，降低使用门槛。

---

#### 2. 批量文档导出

在钉钉文档页面，通过插件提供的界面批量导出：

```
功能入口：插件弹窗 → 批量导出
支持方式：
- 通过文档ID列表批量导出
- 通过当前页面文档列表批量选择导出
- 支持导出数百个文档

格式转换支持：
- .adoc（钉钉文档）→ .md / .docx / .pdf
- .axls（钉钉表格）→ .xlsx
- .adraw（白板）→ .jpg
- .amind（脑图）→ .jpg
```

**价值**：一次性导出数百个文档，节省大量手动操作时间。

---

#### 3. 知识库单个导出

在钉钉知识库页面，一键导出整个知识库：

```
功能入口：插件弹窗 → 导出知识库
导出选项：
- 导出整个知识库（保持目录结构）
- 选择特定文件夹导出
- 支持递归导出子目录
```

**价值**：一键导出整个知识库，保持原有目录结构，便于迁移和归档。
