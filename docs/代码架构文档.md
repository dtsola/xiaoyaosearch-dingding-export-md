## 6. 代码规范

### 6.1 通用规范

**文件命名：**
- Vue组件：PascalCase（如 `DocTree.vue`）
- TypeScript文件：camelCase（如 `useConfig.ts`）
- 样式文件：kebab-case（如 `content.css`）
- 常量文件：camelCase（如 `config.ts`）

**目录结构：**
- 按功能模块划分目录
- 相关文件放在同一目录下
- 公共代码放在 shared 目录

**代码风格：**
- 使用 Prettier 统一代码风格
- 使用 ESLint 保证代码质量
- 编写清晰的注释和文档

### 6.2 Vue 代码规范

**命名约定：**
- 组件：PascalCase（如 `DocTreeItem`）
- 组合式函数：camelCase，以 `use` 开头（如 `useConfig`）
- 变量/函数：camelCase（如 `exportProgress`）
- 常量：UPPER_SNAKE_CASE（如 `MAX_CONCURRENT`）

**组件约定：**
- 单文件组件使用 `<script setup lang="ts">` 语法
- Props 使用 TypeScript 接口定义
- Emits 使用 TypeScript 类型定义

**示例：**
```vue
<script setup lang="ts">
interface Props {
  modelValue: boolean
  title: string
}

interface Emits {
  (e: 'update:modelValue', value: boolean): void
  (e: 'confirm'): void
}

const props = defineProps<Props>()
const emit = defineEmits<Emits>()
</script>
```

### 6.3 TypeScript 代码规范

**命名约定：**
- 接口：PascalCase，无前缀或 `I` 前缀（如 `Config` 或 `IConfig`）
- 类型别名：PascalCase（如 `ExportFormat`）
- 枚举：PascalCase（如 `FileType`）
- 类：PascalCase（如 `ExportManager`）

**类型定义：**
```typescript
// 接口
interface AppConfig {
  exportFormats: ExportFormats
  theme: Theme
}

// 类型别名
type ExportFormat = 'md' | 'docx' | 'pdf'

// 枚举
enum FileType {
  DOC = 'adoc',
  XLS = 'axls',
  DRAW = 'adraw',
  MIND = 'amind',
}
```

### 6.4 CSS/Tailwind 规范

**样式约定：**
- 优先使用 Tailwind 原子类
- 复杂样式使用 `@apply` 或自定义 CSS
- 避免内联样式

**示例：**
```vue
<template>
  <!-- 推荐：使用 Tailwind 类 -->
  <div class="flex items-center justify-between p-4 bg-white rounded-lg shadow-apple">

  <!-- 避免内联样式 -->
  <div style="display: flex; padding: 16px;">
</template>

<style scoped>
/* 复杂样式使用自定义 CSS */
.custom-scroll {
  @apply scrollbar-thin scrollbar-thumb-gray-300 scrollbar-track-gray-100;
}
</style>
```

负责检测钉钉页面类型和注入侧边栏面板。

```typescript
// src/content/index.ts
import { detectPageType } from './pageDetector';
import { injectSidePanel } from './injector';

// 检测页面类型
const pageType = detectPageType(window.location.href);

// 根据页面类型注入相应的功能

  injectSidePanel(pageType);
}
```

### 9.4 Composables 模块

组合式函数，封装可复用的业务逻辑。

```typescript
// src/sidepanel/composables/useDingTalkAPI.ts
export function useDingTalkAPI() {
  const loading = ref(false);
  const error = ref<Error | null>(null);
  const api = new DingTalkAPI();

  const getDocList = async (dentryUuid: string) => {
    loading.value = true;
    try {
      const result = await api.getDocList(dentryUuid);
      return result;
    } catch (e) {
      error.value = e as Error;
      throw e;
    } finally {
      loading.value = false;
    }
  };

  return {
    loading,
    error,
    getDocList,
  };
}
```

## 10. Git 工作流

### 10.1 分支策略

```bash
main          # 生产环境，稳定版本
├── develop   # 开发环境
├── feature/* # 功能分支
├── fix/*     # 修复分支
└── release/* # 发布分支
```

```bash
# 功能开发
```
