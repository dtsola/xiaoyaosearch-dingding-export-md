# 技术方案：小遥搜索钉钉导出工具

## 1. 技术选型

### 前端
- **框架**：自研 Baby 框架（类 Vue.js 响应式框架）
- **UI 库**：Tailwind CSS v4 + DaisyUI v5
- **构建工具**：Vite v8
- **理由**：
  - Baby 框架轻量级（约400行），满足组件化需求
  - Tailwind CSS 提供原子化样式，易于定制 Apple 风格
  - Vite 构建速度快，支持 HMR
  - 复用老项目成熟代码，降低开发风险

### 浏览器扩展
- **类型**：Chrome Extension (Manifest V3)
- **平台**：Chrome / Edge
- **理由**：
  - Manifest V3 是最新标准，安全性更高
  - Chrome/Edge 市场份额最大，覆盖用户最广
  - 支持 File System Access API 实现本地文件操作

### HTTP 请求
- **实现**：自定义 httpRequest 封装（基于 XHR）
- **理由**：
  - XHR 支持上传/下载进度监控
  - 更好的跨域兼容性
  - 可精细控制请求头（钉钉 API 需要特殊认证头）

### 数据流向

```
用户操作流程：
1. 用户访问钉钉文档页面
2. 点击浏览器工具栏的扩展图标
3. background.js 发送消息到 content.js
4. content.js 初始化 Baby 框架和 UI 组件
5. 用户选择文档并点击下载
6. 调用钉钉 API 获取文档内容
7. 调用导出 API 创建导出任务
8. 轮询任务状态直到完成
9. 获取 OSS 下载链接
10. 使用 File System Access API 保存到本地
```

## 3. 数据存储设计

### 本地存储（localStorage）
| 键名 | 类型 | 说明 |
|------|------|------|
| `dddd-export_adoc_as` | string | 文档导出格式（.docx/.md/.pdf） |
| `dddd-export_axls_as` | string | 表格导出格式（.xlsx） |
| `dddd-export_adraw_as` | string | 白板导出格式（.jpg） |
| `dddd-export_amind_as` | string | 脑图导出格式（.jpg） |

## 4. API 接口设计（钉钉 API 调用）

### 认证相关

#### 获取访问令牌
```
POST /portal/api/v1/token/getAccessToken

Headers:
  X-Ajax-Support: content-type

Response:
{
  "accessToken": "xxx",
  "expireTime": 1234567890
}
```

### 文档内容获取

#### 获取文档数据
```
POST /api/document/data

Headers:
  A-Token: <accessToken>
  corp-id: <corpId>
  a-dentry-key: <dentryKey>

Request:
{
  "documentId": "doxcnxxxxx"
}

Response:
{
  "documentContent": {
    "checkpoint": {
      "content": [...]
    }
  }
}
```

### 导出任务相关

#### 上传文档信息
```
POST /core/api/resources/9/upload_info

Headers:
  A-Token: <accessToken>
  corp-id: <corpId>

Request:
{
  "info": {
    "dentryKey": "xxx",
    "docKey": "xxx"
  }
}
```

#### 创建 PDF 导出任务
```
POST /api/v2/files/createExportJob

Headers:
  A-Token: <accessToken>
  corp-id: <corpId>

Request:
{
  "fileType": "pdf",
  "dentryKey": "xxx",
  "docKey": "xxx"
}

Response:
{
  "jobId": "xxx"
}
```

#### 查询导出任务状态
```
POST /api/v2/files/queryExportStatus

Headers:
  A-Token: <accessToken>
  corp-id: <corpId>

Request:
{
  "jobId": "xxx"
}

Response:
{
  "status": "success",
  "downloadUrl": "https://xxx"
}
```

## 6. 技术风险

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| **钉钉 API 变更** | 高 | • 建立用户反馈渠道<br>• 快速响应机制<br>• 版本兼容性处理 |
| **浏览器 API 兼容** | 中 | • 优先支持 Chrome/Edge<br>• 其他浏览器后续跟进<br>• 使用标准 Web API |
| **跨域问题** | 中 | • 使用 XHR 替代 fetch<br>• 正确配置扩展权限<br>• 处理预检请求 |
| **文件系统 API 限制** | 低 | • 提供降级方案（传统下载）<br>• 明确浏览器兼容性 |
| **大量文档性能** | 中 | • 分页加载机制<br>• 并发控制<br>• 进度反馈优化 |
| **用户数据隐私** | 高 | • 所有数据本地处理<br>• 不上传任何文档内容<br>• 明确隐私政策 |

```javascript
// 响应式数据系统

    // 基于 Object.defineProperty 的数据劫持

// 组件系统
class Component {
    created() {}    // 创建时
    mounted() {}    // 挂载后
    // 响应式数据自动更新视图

```

### 主组件 (component/main.js)
- **功能**：主界面渲染、文档列表管理
- **状态**：
  - `spaceId`: 当前空间 ID
  - `docs`: 文档列表
  - `selecteds`: 已选中文档
  - `loading`: 加载状态

### 文档项组件 (component/dentryItem.js)
- **功能**：单个文档/文件夹的渲染和交互
- **递归渲染**：支持文件夹嵌套
- **级联选择**：父节点选中自动选中子节点

## 8. 文件导出流程

### PDF 导出流程
```
1. 获取文档开放令牌 (getOpenToken)
2. 上传文档内容到 OSS (uploadDocument)
3. 创建 PDF 导出任务 (createExportJob)
4. 轮询任务状态 (queryExportStatus)
5. 下载生成的 PDF 文件
```

### Markdown 导出流程
```
1. 获取文档数据 (getDocumentData)
2. 解析文档结构
3. 使用 adoc2md 转换器转换
4. 生成本地 .md 文件
```

### Excel 导出流程
```
1. 获取表格数据
2. 提交导出任务 (submitExportJob)
3. 轮询任务状态
4. 下载 .xlsx 文件
```
