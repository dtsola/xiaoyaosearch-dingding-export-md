# 元数据导出方案

## 目的

导出钉钉文档时，生成元数据映射文件，方便小遥搜索识别文件来源和建立与云端文档的映射关系。

## 数据结构

### manifest.json 格式

```json
{
  "version": "1.0.0",
  "exportTime": "2026-04-08T12:00:00.000Z",
  "source": "dingtalk-docs",
  "exportTool": "xiaoyaosearch-dingding-export",
  "exportToolVersion": "1.0.0",
  "files": [
    {
      "fileName": "项目报告.docx",
      "originalName": "项目报告",
      "dentryUuid": "xxx",
      "dentryKey": "xxx",
      "docKey": "xxx",
      "url": "https://alidocs.dingtalk.com/i/nodes/xxx",
      "fileSize": 102400,
      "extension": "adoc",
      "exportFormat": ".docx",
      "relativePath": "工作文档/项目报告.docx",
      "contentType": "alidoc"
    }
  ]
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `version` | manifest 格式版本 |
| `exportTime` | 导出时间 (ISO 8601) |
| `source` | 数据来源标识 |
| `exportTool` | 导出工具名称 |
| `exportToolVersion` | 导出工具版本 |
| `files[]` | 文件列表 |
| `fileName` | 导出后的文件名 |
| `originalName` | 钉钉文档原始名称 |
| `dentryUuid` | 钉钉文档唯一ID |
| `dentryKey` | 钉钉文档密钥 |
| `docKey` | 文档密钥 |
| `url` | 钉钉文档访问地址 |
| `fileSize` | 文件大小（字节） |
| `extension` | 原始扩展名 |
| `exportFormat` | 导出格式 |
| `relativePath` | 相对路径（含文件名） |
| `contentType` | 内容类型 |

## 实现步骤

### 简化原则

- 不考虑不同知识库导出到同一目录的情况
- 每次导出覆盖/创建新的 `xiaoyaosearch-manifest.xyjson`
- 保持代码简洁，最小化改动

#### 改动位置 2：新增 `writeMetadataFile` 方法（添加到 methods 中，约第 250 行之后）

```javascript
// ============================================
// 新增方法：写入元数据文件
// ============================================
async writeMetadataFile(dirHandle, metadataMap, exportTime) {
    const manifest = {
        version: "1.0.0",
        exportTime: exportTime,
        source: "dingtalk-docs",
        exportTool: "xiaoyaosearch-dingding-export",
        exportToolVersion: version,
        files: metadataMap
    };

    const manifestHandle = await dirHandle.getFileHandle(
        "xiaoyaosearch-manifest.xyjson",
        {create: true}
    );

    const writable = await manifestHandle.createWritable();
    await writable.write(JSON.stringify(manifest, null, 2));
    await writable.close();
}
```

## 小遥搜索使用方式

### 1. 读取 manifest

```javascript
import fs from 'fs';

const manifest = JSON.parse(
    fs.readFileSync('xiaoyaosearch-manifest.xyjson', 'utf-8')
);
```

### 2. 建立索引

```javascript
// 按 dentryUuid 建立索引
const fileMap = new Map();
manifest.files.forEach(file => {
    fileMap.set(file.dentryUuid, file);
});
```

### 3. 匹配云端文档

```javascript
// 用户从钉钉复制文档链接
const dingtalkUrl = "https://alidocs.dingtalk.com/i/nodes/xxx";
const dentryUuid = dingtalkUrl.split('/').pop();

// 查找本地文件
const localFile = fileMap.get(dentryUuid);
if (localFile) {
    console.log(`本地文件: ${localFile.relativePath}`);
    console.log(`导出格式: ${localFile.exportFormat}`);
}
```

### 4. 使用场景

1. **用户复制钉钉链接** → 小遥搜索查找本地文件
2. **用户搜索文档内容** → 小遥搜索显示来源链接
3. **文档更新同步** → 根据 dentryUuid 判断是否需要重新下载

---

## 兼容性考虑

1. **向后兼容**: 老版本没有 manifest，小遥搜索可以检测文件是否存在
2. **每次覆盖**: 重新导出会覆盖旧 manifest（不考虑多知识库合并）
3. **部分导出**: manifest 只记录本次导出的文件
