### 测试 3：导出单个文档

#### 3.1 获取文档 ID

1. 打开钉钉文档（浏览器）
2. 从 URL 中获取文档 ID，格式类似：
   ```
   https://alidocs.dingtalk.com/i/nodes/{nodeId}
   ```

#### 3.2 运行导出命令

```bash
# 替换 <your-doc-id> 为实际文档 ID
npm run dev -- export -d <your-doc-id> -o ./test-output
```

**预期流程：**
1. 检查 Cookie
2. 如果 Cookie 无效，打开浏览器登录
3. 获取文档页面
4. 提取 dentryKey
5. 获取文档内容
6. 解析文档
7. 生成 Markdown
8. 保存文件

**预期结果：**
- 在 `./test-output/` 目录生成 `.md` 文件
- 控制台显示导出进度

### 测试 4：批量导出

#### 4.1 创建测试 ID 文件

```bash
echo "DOC_ID_1" > test-ids.txt
echo "DOC_ID_2" >> test-ids.txt
echo "DOC_ID_3" >> test-ids.txt
```

#### 4.2 运行批量导出

```bash
npm run dev -- docs --file test-ids.txt -o ./test-output
```

**或直接指定 ID：**

```bash
npm run dev -- docs --ids "id1,id2,id3" -o ./test-output
```

**指定并发数：**

```bash
npm run dev -- docs --ids "id1,id2" -o ./test-output -c 2
```

**预期结果：**
- 显示批量导出进度
- 显示成功/失败统计

## 常见问题排查

### 问题 1：Cookie 失效

**现象：** `Cookie 无效或已过期`

**解决：**
```bash
# 删除 Cookie 文件
rm ~/.dingding-export/cookies.json

# 重新运行导出，会自动打开浏览器登录
npm run dev -- export -d <doc-id>
```

### 问题 2：无法提取 dentryKey

**可能原因：**
- 文档 ID 不正确
- 需要登录
- 页面结构变化

**解决：**
- 确认文档 ID 正确
- 确保 Cookie 有效
- 检查是否有权限访问该文档

### 问题 3：网络请求失败

**现象：** `网络请求失败，请检查网络连接`

**解决：**
- 检查网络连接
- 检查防火墙设置
- 尝试使用代理
