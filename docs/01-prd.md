# PRD：小遥搜索钉钉导出工具 MVP版本

> **💡 本工具是 [小遥搜索](https://github.com/dtsola/xiaoyaosearch) 生态的扩展工具**
>
> [小遥搜索](https://github.com/dtsola/xiaoyaosearch) —— 听懂你的话、看懂你的图，用 AI 找到本地任何文件。让搜索像聊天一样简单。

## 1. 产品定位（一句话）

为 **钉钉文档用户** 提供 **批量导出知识库并转换为通用格式** 的 **浏览器插件**

## 1.1 产品目标

本工具是小遥搜索生态的重要组成部分，旨在帮助用户：
1. 将云端钉钉文档备份到本地，配合小遥搜索实现本地 AI 搜索
2. 提供便捷的文档导出功能，降低用户使用门槛
3. 扩大小遥搜索品牌影响力，吸引潜在用户

## 2. 适用人群

- 👨‍💻 **知识管理爱好者** - 希望将云端文档备份到本地，防止数据丢失
- 💼 **自由职业者** - 需要将钉钉内容迁移到 Obsidian、Logseq 等本地笔记工具
- 🏢 **创业团队 IT 运维** - 负责企业知识库的定期备份和合规归档
- 📝 **技术文档维护者** - 需要将 API 文档导出到本地进行版本控制
- 🎓 **学生用户** - 在 Windows/macOS 等不同平台上都需要使用

## 3. 使用场景

### 📦 个人知识备份
将珍贵的钉钉文档备份到本地，即使没有网络也能随时查看，同时避免云端数据丢失风险。

### 🔄 笔记工具迁移
导出为标准 Markdown/.docx/.pdf 等格式，轻松迁移到 Obsidian、Logseq、Notion 等其他知识管理工具。

### 🏛️ 企业知识归档
批量导出整个钉钉知识库，保持原有目录结构，满足企业合规和审计要求。

### 📊 技术文档本地化
将 API 文档、技术规范、设计文档导出到本地，实现离线查阅和版本管理。

### 🚀 离职文档交接
一键导出所有参与过的项目文档，快速完成工作资料交接，避免数据丢失。

## 5. 功能清单（MVP范围）

### P0功能（必须有）
- [x] **插件安装与激活**：支持 Chrome/Edge 浏览器扩展安装
- [x] **页面检测与注入**：自动检测钉钉文档页面并注入插件功能
- [x] **文档列表加载**：获取并展示当前页面的文档/文件夹列表
- [x] **单选/多选功能**：支持选择单个或多个文档进行导出
- [x] **文件夹级联选择**：选择文件夹时自动选中所有子项
- [x] **批量下载**：支持同时下载多个选中的文档
- [x] **格式转换**：

- [x] **导出格式配置**：用户可自定义各类型文档的默认导出格式

## 6. 核心流程图

### 插件激活流程
```
用户访问钉钉文档 → 点击扩展图标 → 注入内容脚本
→ 检测页面类型 → 加载文档列表 → 显示插件界面
```

### 文档导出流程
```
用户选择文档/文件夹 → 点击"下载选中"
→ 选择本地保存目录 → 递归处理选中项
→ 调用钉钉导出API → 轮询任务状态 → 下载到本地
→ 更新进度显示 → 完成/失败提示
```

### API调用流程
```
获取认证信息 → 获取文档列表 → 用户选择文档
→ 上传文档元数据 → 创建导出任务 → 轮询任务状态
→ 获取OSS下载链接 → 写入本地文件
```

## 8. 数据埋点计划

| 事件 | 触发时机 | 关键参数 |
|------|---------|---------|
| `plugin_activated` | 用户点击插件图标激活 | page_type, url |
| `document_list_loaded` | 文档列表加载完成 | doc_count, load_time |
| `document_selected` | 用户选择/取消选择文档 | doc_type, is_folder |
| `download_started` | 用户点击下载按钮 | selected_count, total_size |
| `export_format_changed` | 用户修改导出格式设置 | doc_type, new_format |
| `download_completed` | 单个文档下载完成 | doc_type, format, size, time |
| `download_failed` | 文档下载失败 | doc_type, error_code |
| `batch_download_finished` | 批量下载全部完成 | success_count, fail_count |
| `settings_opened` | 打开设置页面 | source |
| `help_opened` | 打开帮助页面 | source |
| `error_shown` | 显示错误提示 | error_type, error_msg |

## 9. 非功能需求

### 性能要求
- **插件激活响应**：< 500ms
- **文档列表加载**：< 2秒（100个文档以内）
- **格式转换**：根据文件大小，一般文档 < 30秒
- **内存占用**：< 200MB

### 兼容性要求
- **浏览器**：Chrome 90+, Edge 90+
- **操作系统**：Windows, macOS, Linux
- **钉钉域名**：
  - alidocs.dingtalk.com
  - docs.dingtalk.com

### 安全要求
- **数据存储**：所有配置仅存储在本地，不上传服务器
- **权限控制**：仅请求必要的浏览数据权限
- **隐私保护**：不收集用户文档内容
- **通信安全**：使用 HTTPS 与钉钉服务器通信

### 可靠性要求
- **导出成功率**：> 95%
- **崩溃率**：< 1%
- **错误恢复**：网络中断后支持重试

### 可维护性要求
- **代码规范**：遵循项目编码规范
- **日志记录**：关键操作记录日志便于调试
- **版本管理**：语义化版本号
