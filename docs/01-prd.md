# PRD：小遥搜索钉钉导出工具 MVP版本

## 1. 产品定位（一句话）
为 **使用钉钉进行知识管理的用户** 提供 **一键将钉钉文档/知识库导出为 Markdown 格式** 的 **跨平台命令行工具**，便于对接小遥搜索实现本地 AI 智能搜索。

## 2. 用户故事（核心场景）

**场景1：个人知识备份**
- 作为 **知识管理爱好者**
- 我想要 **将钉钉中的个人文档批量导出到本地**
- 以便于 **防止云端数据丢失，并能离线查看**

**场景2：笔记工具迁移**
- 作为 **自由职业者/Indie Hacker**
- 我想要 **将钉钉文档导出为标准 Markdown 格式**
- 以便于 **迁移到 Obsidian、Logseq 等本地笔记工具**

**场景3：对接小遥搜索**
- 作为 **小遥搜索用户**
- 我想要 **将钉钉知识库导出后接入小遥搜索**
- 以便于 **实现私有知识库的本地 AI 智能搜索**

**场景4：企业知识归档**
- 作为 **创业团队 IT 运维**
- 我想要 **定期自动备份企业钉钉知识库**
- 以便于 **满足合规要求和数据安全**

**场景5：自动化集成**
- 作为 **DevOps 工程师**
- 我想要 **将导出工具集成到 CI/CD 流程**
- 以便于 **实现无人值守的自动化知识库备份**

## 3. 功能清单（MVP范围）

### P0功能（必须有）

#### 配置管理
- [ ] **交互式初始化** (`dingding-export init`)
  - 引导用户输入钉钉应用的 AppKey、AppSecret
  - 设置默认输出目录
  - 支持多配置文件（profile），便于管理多个账号/企业

#### 单个文档导出
- [ ] **导出单个文档** (`dingding-export export -d <doc_id>`)
  - 支持通过文档 ID 导出
  - 支持通过文档 URL 导出（自动提取 ID）
  - 自动下载文档中的图片
  - 导出为标准 Markdown 格式

#### 批量文档导出
- [ ] **批量导出** (`dingding-export docs`)
  - 支持从文件读取文档 ID 列表（`--file <path>`）
  - 支持命令行逗号分隔的 ID 列表（`--ids <list>`）
  - 显示实时导出进度
  - 支持并发控制（`--concurrency <n>`，默认5）

#### 知识库导出
- [ ] **知识库文档导出** (`dingding-export export -w <wiki_id>`)
  - 支持导出知识库中的单个文档
  - 保持知识库的目录结构

- [ ] **完整知识库导出** (`dingding-export wiki <wiki_id>`)
  - 递归导出整个知识库
  - 保持原有目录结构和层级关系
  - 支持设置最大递归深度

#### 配置管理命令
- [ ] **配置管理** (`dingding-export config`)
  - `config get [key]` - 获取配置项
  - `config set <key> <value>` - 设置配置项
  - `config list` - 列出所有配置
  - `config use <profile>` - 切换配置文件

### P1功能（最好有）

#### 增量导出
- [ ] **增量导出模式** (`--incremental`)
  - 仅导出有更新的文档
  - 基于本地文件的修改时间判断

#### 文件夹导出
- [ ] **文件夹导出** (`dingding-export folder <folder_id>`)
  - 递归导出文件夹中的所有文档
  - 支持设置最大递归深度

#### 附件下载
- [ ] **附件下载**
  - 自动下载文档中的附件文件
  - 支持关闭附件下载（`--no-assets`）

#### 错误处理
- [ ] **错误恢复**
  - 导出失败时记录日志
  - 支持断点续传（记录已导出的文档）

### P2功能（未来考虑）

- [ ] **只生成索引** (`--index-only`) - 仅生成知识库索引文件，不下载文档内容
- [ ] **环境变量支持** - 通过环境变量设置配置项
- [ ] **导出模板自定义** - 支持自定义 Markdown 导出模板
- [ ] **多格式导出** - 支持 PDF、HTML 等格式
- [ ] **GUI 界面** - 提供图形化操作界面

## 4. 核心流程图

```
用户安装 npm install -g xiaoyaosearch-dingding-export
         ↓
用户运行 dingding-export init
         ↓
交互式配置 AppKey、AppSecret、输出目录
         ↓
配置保存到 ~/.dingding-export/config.json
         ↓
用户执行导出命令
         ↓
┌────────────────┬────────────────┬────────────────┐
│  export -d     │  docs --file   │  wiki <id>     │
│  单个文档      │  批量导出      │  知识库        │
└────────────────┴────────────────┴────────────────┘
         ↓
    调用钉钉 API
         ↓
    转换为 Markdown
         ↓
    下载图片/附件
         ↓
    保存到本地
         ↓
对接小遥搜索实现本地 AI 智能检索
```

## 5. 命令结构

```
dingding-export
├── init [--profile <name>]          # 初始化配置
├── export (-d <id> | -w <id>)       # 导出单个文档
│   └── [-o <dir>] [--no-images] [--incremental]
├── docs                             # 批量导出文档
│   ├── --file <path> | --ids <list>
│   └── [-o <dir>] [-c <n>] [--incremental]
├── folder <folder_id>               # 导出文件夹
│   └── [-o <dir>] [--depth <n>]
├── wiki <wiki_id>                   # 导出完整知识库
│   └── [-o <dir>] [--depth <n>] [--index-only]
└── config                           # 配置管理
    ├── get [key]
    ├── set <key> <value>
    ├── list
    └── use <profile>
```

## 6. 数据埋点计划

| 事件 | 触发时机 | 关键参数 | 用途 |
|------|---------|---------|------|
| `init` | 用户初始化配置 | profile_name | 统计新增用户 |
| `export_doc` | 导出单个文档 | doc_id, has_images | 统计单篇导出 |
| `export_docs` | 批量导出文档 | count, success_count, fail_count | 统计批量导出 |
| `export_wiki` | 导出知识库 | wiki_id, depth | 统计知识库导出 |
| `error` | 导出失败 | error_code, error_message | 错误监控 |

**埋点实现：** 使用开源分析工具（如 Google Analytics 4MP 或自建简单统计）

## 7. 非功能需求

### 性能要求
- 单个文档导出响应时间 < 5秒
- 批量导出 100 篇文档 < 5分钟
- 并发请求可配置，默认 5，最大不超过 20

### 兼容性
- **Node.js**: >= 18.0.0
- **操作系统**: Windows、macOS、Linux
- **包管理器**: npm、pnpm、yarn

### 安全要求
- 配置文件存储在用户目录，权限设置为仅所有者可读写
- AppSecret 不在日志中输出
- 支持 `.gitignore` 忽略配置文件，防止意外提交

### 可用性
- 提供详细的帮助信息（`--help`）
- 错误提示清晰，包含解决建议
- 支持 `--debug` 模式输出详细日志

### 可维护性
- TypeScript 编写，类型安全
- 模块化设计，便于扩展
- 单元测试覆盖率 > 80%

## 9. 配置文件结构

配置文件位置：`~/.dingding-export/config.json`

```json
{
  "profiles": {
    "default": {
      "appId": "dingxxxxxxxxx",
      "appSecret": "xxxxxxxxxxxxxxxx",
      "endpoint": "https://api.dingtalk.com",
      "outputDir": "./output",
      "concurrency": 5,
      "downloadImages": true,
      "downloadAssets": true
    }
  },
  "currentProfile": "default"
}
```
