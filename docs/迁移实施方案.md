## 1. 迁移背景

### 1.1 当前状态
| 项目 | 说明 |
|------|------|
| **构建工具** | Webpack 5.94.0 |
| **代码风格** | CommonJS (`require`/`module.exports`) |
| **模块系统** | 自研 Baby 框架（类 Vue） |
| **样式方案** | Tailwind CSS v4 + DaisyUI |
| **调试模式** | 书签脚本注入模式 |

### 1.2 迁移动机
- ✅ 开发服务器启动更快
- ✅ HMR（热模块替换）响应更迅速
- ✅ 配置更简洁，维护成本更低
- ✅ 原生 ESM 支持，符合现代前端标准

### 1.3 可行性评估
| 评估项 | 难度 | 说明 |
|--------|------|------|
| **CommonJS → ESM 转换** | ⭐⭐⭐ | 需要修改 10+ 个模块文件 |
| **EJS 模板处理** | ⭐⭐ | 需要引入 Vite EJS 插件或转换为纯 HTML |
| **多入口配置** | ⭐ | Vite 原生支持 |
| **书签脚本模式** | ⭐⭐⭐ | 需要适配输出路径和模块加载方式 |
| **内联资源（生产）** | ⭐⭐ | 需要额外插件实现 |

**总体难度：⭐⭐⭐ (中等)**

### 2.2 Vite 配置文件

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import { viteCommonjs } from '@originjs/vite-plugin-commonjs';
import { createHtmlPlugin } from 'vite-plugin-html';
import { viteStaticCopy } from 'vite-plugin-static-copy';

export default defineConfig(({ mode }) => ({
  plugins: [
    // CommonJS 兼容
    viteCommonjs(),

    // HTML 模板处理
    createHtmlPlugin({
      minify: mode === 'production',
      pages: [
        {
          entry: '/src/app.js',
          template: '/src/app.ejs',
          filename: 'index.html',
          inject: true,
        },
      ],
    }),

    // 静态资源复制（书签脚本模式）
    viteStaticCopy({
      targets: [
        {
          src: 'dist/assets/*.js',
          dest: '../',
          rename: (file) => `main.js`,
        },
      ],
    }),
  ],

  // 构建配置
  build: {
    outDir: 'dist',
    assetsDir: 'assets',
    rollupOptions: {
      input: {
        main: './src/app.js',
      },
      output: {
        entryFileNames: 'assets/[name].[hash].js',
        chunkFileNames: 'assets/[name].[hash].js',
        assetFileNames: 'assets/[name].[hash].[ext]',
        // 书签脚本模式：生成独立 JS 文件
        ...(
          mode === 'development' ? {
            entryFileNames: 'main.js',
            format: 'iife',
          } : {}
        ),
      },
    },
  },

  // 开发服务器
  server: {
    port: 56860,
    open: true,
    hmr: true,
  },

  // CSS 配置
  css: {
    postcss: './postcss.config.js',
    devSourcemap: true,
  },

  // 环境变量
  define: {
    'process.env.NODE_ENV': JSON.stringify(mode),
  },
}));
```

### 2.3 CommonJS → ESM 转换策略

#### 转换示例

**转换前（CommonJS）：**
```javascript
const Baby = require("./script/index");
const { getCfg, setCfg } = require("./script/cfg");

module.exports = {
    getCfg,
    setCfg
};
```

**转换后（ESM）：**
```javascript
import Baby from "./script/index.js";
import { getCfg, setCfg } from "./script/cfg.js";

export {
    getCfg,
    setCfg
};
```

### 阶段三：配置适配（预估 0.5 天）

#### Step 3.1：创建 Vite 配置
创建 [vite.config.js](vite.config.js)，参考 2.2 节配置。

#### Step 3.2：处理 EJS 模板

**方案 A：使用 vite-plugin-html（推荐）**
```javascript
import { createHtmlPlugin } from 'vite-plugin-html';

createHtmlPlugin({
  minify: mode === 'production',
  pages: [{
    entry: '/src/app.js',
    template: '/src/app.ejs',
    filename: 'index.html',
  }],
})
```

**方案 B：转换为纯 HTML + 模板变量**
如 EJS 逻辑简单，可直接转换为纯 HTML。

#### Step 3.3：适配书签脚本模式

开发模式下，生成可被书签注入的 `main.js`：

```javascript
// vite.config.js - 开发模式特殊配置
export default defineConfig(({ mode }) => ({
  build: mode === 'development' ? {
    rollupOptions: {
      output: {
        entryFileNames: 'main.js',
        format: 'iife',
      }
    }
  } : {}
}));
```

更新 [app.ejs](src/app.ejs) 中的书签链接：

## 4. 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| **CommonJS 循环依赖** | 高 | 中 | 使用 Vite CommonJS 插件；必要时重构模块结构 |
| **书签脚本模式失效** | 高 | 中 | 保留 IIFE 格式输出；单独配置开发构建 |
| **EJS 模板复杂逻辑** | 中 | 低 | 使用 vite-plugin-html 或简化模板逻辑 |
| **第三方库不兼容** | 中 | 低 | 使用 @vitejs/plugin-legacy 兼容旧浏览器 |
| **构建产物差异** | 中 | 低 | 充分测试生产构建；必要时调整 Rollup 配置 |

## 5. 回滚方案

如迁移过程中遇到无法解决的问题，执行回滚：

```bash
# 1. 切回主分支
git checkout main

# 2. 删除迁移分支
git branch -D feat/migrate-to-vite

# 3. 恢复依赖
cp .backup/package.json.backup package.json
rm -rf node_modules package-lock.json
npm install

# 4. 恢复配置
cp .backup/webpack.config.js webpack.config.js
rm vite.config.js
```
