## 1. 总体技术架构

### 架构模式
- **架构类型**：浏览器扩展（Chrome Extension Manifest V3）
- **部署方式**：Chrome Web Store / Edge Add-ons / GitHub Releases
- **核心特性**：
  - 纯前端方案，无需后端服务器
  - 在钉钉文档页面注入侧边栏面板
  - 使用钉钉页面已有Cookie进行API调用
  - 所有数据存储在用户本地浏览器
- **模块化设计**：
  - Content Script（页面注入）
  - Side Panel（侧边栏UI）
  - Background Service Worker（后台服务）
  - Shared Modules（共享代码）

## 2. 前端技术选型

### 2.1 核心框架

#### Vue 3.4+ ✅

**选择原因：**
- 与飞书导出工具保持技术栈一致，便于维护和知识复用
- 组合式API（Composition API）提供更好的逻辑复用
- 响应式系统性能优异，适合处理文档树状态
- 轻量级（~34KB gzipped），适合浏览器扩展场景

**核心优势：**
1. **组合式API** - 逻辑复用能力强，适合提取可复用的业务逻辑
2. **优秀的性能** - 编译时优化，静态提升，更小的运行时开销
3. **TypeScript支持** - 原生TypeScript支持，类型推导完善
4. **生态成熟** - VueUse、Pinia等周边库完善
5. **学习曲线平缓** - 与Vue 2相似，迁移成本低

**替代方案对比：**

| 方案 | 优势 | 劣势 | 结论 |
|------|------|------|------|
| **Vue 3** ✅ | 轻量、性能好、与飞书工具一致 | 生态小于React | **选择** |
| React 18 | 生态更大、组件库丰富 | 包体积较大、需要额外状态库 | 不选 |
| Svelte 4 | 编译时框架、包体积最小 | 生态较小、学习成本 | 不选 |
| Vanilla JS | 无依赖、最小体积 | 大型项目维护困难 | 不选 |

#### Vite 5.0+ ✅

**选择原因：**
- Vue官方推荐构建工具，开箱即用
- 极速的热更新（HMR），开发体验极佳
- 原生ES模块支持，无需打包即可开发
- 优化的生产构建，自动代码分割

**核心优势：**
1. **极速启动** - 冷启动速度秒级
2. **HMR** - 热更新速度在毫秒级
3. **TypeScript支持** - 开箱即用，无需额外配置
4. **插件生态** - CRXJS等插件简化扩展开发

### 2.2 UI组件库

#### Tailwind CSS 3.4+ ✅

**选择原因：**
- 原子化CSS，可以快速构建苹果风格UI
- 高度可定制，完全控制设计细节
- 生产构建自动清除未使用的样式，包体积小
- 与Vue配合良好

**核心优势：**
1. **原子化** - 每个类对应一个CSS属性
2. **可定制** - 通过配置文件轻松定制主题
3. **按需生成** - 仅生成使用的样式
4. **响应式** - 内置响应式工具类
5. **暗色模式** - 内置暗色模式支持

**替代方案对比：**

| 方案 | 优势 | 劣势 | 结论 |
|------|------|------|------|
| **Tailwind CSS** ✅ | 高度可定制、苹果风格易实现 | 需要学习原子类 | **选择** |
| Element Plus | 组件丰富、开箱即用 | 设计风格固定、包体积大 | 不选 |
| Ant Design Vue | 组件丰富、企业级 | 设计风格固定、包体积大 | 不选 |
| Naive UI | 组件设计现代 | 生态较小 | 不选 |
| 纯CSS | 完全控制、无依赖 | 开发效率低 | 不选 |

### 2.3 状态管理

#### Pinia 2.1+ ✅

**选择原因：**
- Vue 3官方推荐的状态管理库
- API简洁直观，学习成本低
- 完善的TypeScript支持
- 支持DevTools调试

**核心优势：**
1. **简洁API** - 去除了Vuex的mutation，只有state、getters、actions
2. **TypeScript** - 完善的类型推导
3. **模块化** - 自动代码分割，每个store独立
4. **DevTools** - 支持Vue DevTools调试

**使用场景：**
- 侧边栏展开/收起状态
- 导出任务队列状态
- 用户配置状态

**替代方案对比：**

| 方案 | 优势 | 劣势 | 结论 |
|------|------|------|------|
| **Pinia** ✅ | 官方推荐、API简洁 | 较新，生态较小 | **选择** |
| Vuex 4 | 成熟稳定、生态大 | API复杂、冗余mutation | 不选 |
| 组合式函数 | 轻量、简单 | 大型应用状态管理困难 | 不选 |

### 2.4 网络通信

#### 原生 Fetch API ✅

**选择原因：**
- 浏览器原生支持，无需额外依赖
- 在Content Script中调用钉钉API，不受CORS限制
- 使用钉钉页面已有Cookie，无需额外认证

**核心优势：**
1. **原生支持** - 无需引入额外库
2. **同域请求** - 在钉钉页面内调用，无跨域问题
3. **流式支持** - 支持流式响应（暂不需要）

### 2.5 其他前端库

#### VueUse 10.7+ ✅

**选择原因：**
- 丰富的组合式API工具库
- 浏览器API封装（Storage、Clipboard等）
- 与Vue 3完美配合

**核心优势：**
1. **丰富工具** - 200+ 组合式函数
2. **类型完善** - TypeScript支持完善
3. **按需引入** - Tree-shaking友好

**使用场景：**
- `useStorage()` - LocalStorage响应式存储
- `useClipboard()` - 剪贴板操作
- `useDebounceFn()` - 防抖函数
- `useThrottleFn()` - 节流函数

#### CRXJS 2.0+ ✅

**选择原因：**
- Vite官方推荐的Chrome扩展插件
- 简化Manifest V3扩展开发
- 支持HMR，开发体验好

**核心优势：**
1. **Vite集成** - 与Vite无缝集成
2. **HMR支持** - 修改代码后自动刷新扩展
3. **TypeScript** - 完善的类型支持

## 5. 存储技术选型

### 5.1 浏览器本地存储

#### LocalStorage ✅

**选择原因：**
- 浏览器原生支持，无需额外依赖
- 容量足够（5-10MB）
- API简单，同步访问

**技术规格：**
- **容量**：5-10MB
- **访问**：同步API
- **持久性**：永久（除非用户清除）
- **作用域**：同源下共享

**核心优势：**
1. **简单易用** - API简单
2. **同步访问** - 无需async/await
3. **足够容量** - 配置和历史记录足够

**使用场景：**
- 用户配置（导出格式、主题等）
- 导出历史记录
- 首次使用标记

#### Chrome Storage API ✅

**选择原因：**
- 跨标签页同步配置
- 支持同步到用户账号（可选）

**技术规格：**
- **容量**：local约10MB，sync约100KB
- **访问**：异步API
- **持久性**：永久

**核心优势：**
1. **跨标签** - 多标签页共享状态
2. **可同步** - 可选择同步到云端
3. **监听变化** - 监听存储变化事件

**使用场景：**
- 跨标签页的配置同步
- 导出任务状态共享

#### IndexedDB（可选）

**选择原因：**
- 大容量存储
- 适合缓存API响应

**技术规格：**
- **容量**：通常50%以上磁盘空间
- **访问**：异步API
- **性能** - 适合大量数据

**使用场景：**
- API响应缓存（未来考虑）
- 大型文档树缓存（未来考虑）

|------|------|------|---------|

| **Chrome Storage** ✅ | 跨标签、可同步 | 异步API | 跨标签同步 |
| IndexedDB | 大容量、高性能 | API复杂 | 大量数据 |

## 10. 技术选型总结

### 核心原则
1. **与飞书导出工具一致**：便于维护和知识复用
2. **轻量高效**：适合浏览器扩展场景
3. **隐私优先**：不收集用户数据
4. **苹果风格设计**：Tailwind CSS实现
5. **成熟稳定**：选择成熟稳定的技术栈

### 技术亮点
✅ **Vue 3组合式API**：逻辑复用能力强
✅ **TypeScript**：类型安全，减少运行时错误
✅ **Tailwind CSS**：苹果风格UI设计
✅ **Vite**：极速开发体验
✅ **纯前端方案**：零服务器成本
✅ **无第三方依赖**：保护用户隐私
✅ **开源透明**：代码完全开源
