## 一、公众号文章（同步其他文章平台）

### 1.1 标题

```
小遥搜索生态新成员：一键导出钉钉文档，实现本地AI搜索
```

### 1.2 正文结构

#### 背景故事

作为一名独立开发者，我之前开发了 [小遥搜索](https://github.com/dtsola/xiaoyaosearch)——一个本地 AI 搜索工具，目前已经获得了 1000+ GitHub Stars。

> **小遥搜索：听懂你的话、看懂你的图，用 AI 找到本地任何文件。让搜索像聊天一样简单。**

发布后，收到很多用户反馈，其中一条让我印象特别深刻：

**"能搜索钉钉文档吗？我的技术文档都在钉钉上。"**

我调研了一下，发现问题很明显：

**钉钉文档存储在云端，小遥搜索作为本地工具，无法直接访问。**

用户想要用 AI 搜索钉钉文档，只能先手动下载到本地。
但钉钉官方只能一篇篇操作，太慢了！

我算了下，300篇文档手动导出至少需要一整天。

**所以问题变成了：如何高效地把钉钉文档导出到本地，然后配合小遥搜索实现AI搜索？**

作为小遥搜索的生态扩展工具，这个导出工具的核心价值就是：
1. **高效导出**：批量导出钉钉文档到本地
2. **AI搜索**：导出后配合小遥搜索，用自然语言秒级找到任何内容

当然，附加的价值是实现了数据备份——对于重视知识管理的人来说，云端数据一定要有本地备份，这是数据安全的基本常识。

**那就自己写一个吧！**

作为独立开发者，遇到问题就解决问题，这是我的日常。

我开发这个工具的价值有两个：
1. **数据安全**：定期备份钉钉文档到本地，防止云端风险
2. **AI 搜索**：导出的文档可直接导入小遥搜索，实现自然语言搜索

#### 解决方案

所以我开发了这个工具——一个浏览器扩展插件，无需安装任何软件，打开就能用。

核心功能：
✅ 单个文档/批量文档导出
✅ 文件夹递归导出
✅ 完整知识库导出
✅ 多格式转换（.docx/.md/.pdf/.xlsx/.jpg）
✅ 目录结构保持
✅ 实时导出进度显示
✅ 跨平台支持（Windows/macOS/Linux）

使用超简单：
1. 打开浏览器，访问 `chrome://extensions/`
2. 开启"开发者模式"
3. 加载已解压的扩展程序
4. 打开钉钉文档，点击插件图标
5. 勾选需要导出的文档，点击"下载选中"
